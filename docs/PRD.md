# 제품 요구사항 정의서 (PRD) - 마이링크 (MyLink)

## 1. 프로젝트 개요
* **프로젝트명:** 마이링크 (MyLink)
* **목적:** 여러 개의 링크(SNS, 포트폴리오, 블로그, 프로젝트 등)를 하나의 통합된 페이지로 모아서 공유할 수 있는 '멀티 링크 프로필' 서비스 제공
* **대상 사용자 (타겟):** 
  * 개발자 (GitHub, 기술 블로그, 포트폴리오 링크 통합)
  * 크리에이터 (유튜브, 인스타그램, 커뮤니티 링크 통합)

## 2. 핵심 기능 목록

### 필수 기능 (MVP)
1. **회원가입 및 로그인:** Firebase를 활용한 Google 소셜 로그인 단일 지원
2. **프로필 관리:** 닉네임(displayName), 실제 이름(username), 한줄 소개 인라인 편집(Inline Edit) 및 수정
3. **링크 관리 (CRUD):** 새로운 링크 추가, 수정(인라인 편집), 삭제 기능 (Google API를 활용한 파비콘 추출)
4. **프로필 퍼블리싱 (공유):** `mylink.com/displayName` 형태의 고유 URL 제공 및 접속 기능

### 제외 및 추후 기능
* **링크 클릭 조회수:** 각 링크별 클릭 횟수 기록 (추후 업데이트 예정)

## 3. 각 기능의 상세 설명

### 3.1 회원가입 및 로그인
* **설명:** 사용자가 본인만의 프로필을 관리하기 위해 계정을 생성합니다.
* **상세:** 
  * Firebase Authentication을 이용한 구글 계정 연동(Google OAuth) 전용
  * 구글 로그인 시, 지메일 계정의 앞부분(예: `abc@gmail.com`의 `abc`)을 자동으로 추출하여 **displayName**의 초기값으로 설정

### 3.2 프로필 관리
* **설명:** 프로필 페이지 최상단에 노출되는 사용자의 핵심 정보를 관리합니다.
* **상세:**
  * 프로필 이미지 업로드 기능 없이 **닉네임(displayName, URL 슬러그로 사용됨), 실제 이름(username), 짧은 자기소개(Bio)** 입력 및 관리
  * 닉네임, 실제 이름, 소개글 모두 텍스트를 클릭하여 그 자리에서 즉시 변경하는 **인라인 편집(Inline Editing)** 방식으로 수정 가능해야 함

### 3.3 링크 관리
* **설명:** 사용자가 공유하고자 하는 다수의 URL을 추가하고 관리합니다.
* **상세:**
  * 링크 제목(타이틀)과 실제 연결될 대상 URL 추가, 삭제 기능
  * 링크 제목과 URL 수정 시 폼(Form) 이동 없이 **인라인 편집**을 통해 수정
  * 대상 URL 도메인을 기반으로 **구글 파비콘 API(Google Favicon API)**를 활용하여 링크 아이템 좌측에 해당 사이트의 아이콘을 자동 출력
  * 방문자가 링크 클릭 시 새 탭(target="_blank")에서 열리도록 처리

### 3.4 UI 디자인 시스템
* **설명:** 애플리케이션의 시각적 요소를 정의합니다.
* **상세:**
  * UI 컴포넌트로 `shadcn/ui` 라이브러리를 전면 도입하여 일관되고 모던한 인터페이스 구성
  * 별도의 복잡한 테마(Theme) 설정 기능은 제공하지 않음

### 3.5 프로필 퍼블리싱
* **설명:** 편집한 프로필을 실제 배포하여 공유 가능한 상태로 만듭니다.
* **상세:**
  * 가입 시 별도의 핸들(ID)을 만들지 않고 사용자의 **displayName**을 직접 URL 슬러그(Slug)로 사용하여 `mylink.com/displayName` 형태로 구성
  * '공유하기' 버튼을 통한 URL 클립보드 자동 복사 및 간단한 팝업 알림 제공

## 4. 데이터베이스 모델링 요구사항
* 데이터베이스는 **Firebase Firestore** 기반으로 설계합니다.
* 사용자 문서 내에 `displayName`(닉네임 및 URL용)과 `username`(실제 사용자의 이름) 필드가 반드시 포함되어야 합니다.
* 링크 리스트 등 관련된 데이터는 단일 문서의 배열(Array)이 아닌 **서브 컬렉션(Sub-collection)**을 사용하여 구조화합니다.

## 5. 비기능적 요구사항 (Non-Functional Requirements)

### 5.1 성능 (Performance)
* 불특정 다수가 접속하는 프로필 페이지는 초기 로딩 속도(FCP)가 1.5초 이내여야 합니다. (Next.js의 SSR 또는 SSG 활용)

### 5.2 보안 및 권한 (Security)
* 사용자는 로그인한 본인의 계정으로 생성된 프로필과 링크만 수정 및 삭제할 수 있어야 합니다.
* XSS 공격 방지를 위해 사용자가 입력하는 닉네임, 실제 이름, 소개글, 링크 URL에 대한 철저한 검증 및 이스케이프 처리가 필수적입니다.

### 5.3 호환성 (Compatibility)
* 주요 모던 브라우저 (Chrome, Safari, Edge, Firefox) 최신 버전에서 정상 작동해야 합니다.
* 모바일 기기 접속 비율이 압도적으로 높으므로 모바일 화면 렌더링(Mobile-first)이 완벽하게 지원되어야 합니다.
